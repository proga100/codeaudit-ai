**1. HealthScore (`health-score.tsx`)** — Per COMP-06:
```
"use client";
```
Props: `score: number`, `size?: "sm" | "lg"`, `className?: string`

Implementation (convert from prototype lines 197-222):
- Letter grade: A (>=90), B (>=80), C (>=70), D (>=50), F (<50)
- Color by threshold: score > 70 = `var(--success)`, score > 40 = `var(--warning)`, else `var(--destructive)`
- Size "lg": 110px diameter, strokeWidth 6, shows score number (28px, font-bold) + grade letter (13px, text-muted)
- Size "sm": 56px diameter, strokeWidth 4, shows score number (16px, font-bold) only (no grade letter)
- SVG ring: background circle uses `stroke="var(--border)"`, progress arc uses computed color
- Ring math: `r = (size - strokeWidth) / 2`, `circumference = 2 * PI * r`, `offset = circumference - (score / 100) * circumference`
- SVG rotated -90deg so arc starts from top
- Score text centered with absolute positioning inside the SVG container
- Progress arc has `transition: stroke-dashoffset 1s ease` for animation

Use inline `style` for SVG stroke colors since they reference CSS variables dynamically computed. Use Tailwind for layout classes.

**2. SeverityBar (`severity-bar.tsx`)** — Per COMP-07:
```
"use client";
```
Props: `data: { critical: number; high: number; medium: number; low: number; info: number }`, `className?: string`

Implementation (convert from prototype lines 224-246):
- 5 bars in a flex row, aligned to bottom (`items-end`), gap-3, height 100px
- Each bar column: count label (12px, font-semibold, colored by severity) -> colored bar -> severity label (10px, uppercase, tracking-wide, text-muted)
- Bar height: proportional — `maxVal = Math.max(...Object.values(data), 1)`, bar height = `Math.max((value / maxVal) * 80, 4)` px
- Bar width: flex-1, max-width 36px, rounded-md (6px), opacity 85%
- Bar transition: `transition: height 0.6s ease`

Severity color mapping (use CSS variables):
- critical: `var(--severity-critical)` (#ef4444)
- high: `var(--severity-high)` (#f97316)
- medium: `var(--severity-medium)` (#eab308)
- low: `var(--severity-low)` (#3b82f6)
- info: `var(--severity-info)` (#71717a)

Labels: "CRIT", "HIGH", "MED", "LOW", "INFO" (first 4 chars of each, uppercase)

Use inline `style` for dynamic bar heights and severity colors (since they vary per bar).

**3. Modal (`modal.tsx`)** — Per COMP-08:
```
"use client";
```
Props: `open: boolean`, `onClose: () => void`, `children: React.ReactNode`, `className?: string`

Implementation (convert from prototype lines 248-270):
- When `open=false`: render nothing (return null)
- Fixed overlay: `fixed inset-0 z-[1000] flex items-center justify-center`
- Backdrop: `absolute inset-0 bg-black/60 backdrop-blur-[8px]` — clicking backdrop calls `onClose`
- Content panel: `relative bg-surface border border-border rounded-[--radius-modal] p-7 max-w-[480px] w-[90%] shadow-[0_24px_48px_rgba(0,0,0,0.3)] animate-fade-in`
- The content panel has the `animate-fade-in` class for entry animation

Do NOT use Radix Dialog for this — the prototype uses a simple div-based modal. Using Radix would add unnecessary complexity and deviation from the spec. Keep it simple: conditional render + backdrop + content div.

Add `useEffect` to handle Escape key press → calls `onClose`. Also add `useEffect` to lock body scroll when open (`document.body.style.overflow = "hidden"`, restore on cleanup).
