  <action>
    **Server component (page.tsx):**
    Follow the exact pattern from audit/[id]/page.tsx (Phase 7 established pattern):
    - Accept `params: Promise<{ id: string }>`, await the id
    - Use `getDb()` to query `audits` table with `eq(audits.id, id)` -- call `.get()` for single record
    - If not found, call `notFound()`
    - If audit.status !== "completed" or audit.findings is null, redirect to `/audit/${id}` (progress page)
    - Query `auditPhases` table: `db.select().from(auditPhases).where(eq(auditPhases.auditId, id)).orderBy(auditPhases.phaseNumber).all()`
    - Serialize the audit data for client: id, folderName, folderPath, auditType, depth, findings (the full AuditFindings JSON), tokenCount, actualCostMicrodollars, completedAt (ISO string), startedAt (ISO string)
    - Serialize auditPhases: array of { phaseNumber, status, findings (AuditFinding[] array), tokensUsed, startedAt (ISO), completedAt (ISO) }
    - Pass serialized data to `<ResultsDashboard audit={serialized} phases={serializedPhases} />`

    **Client component (results-dashboard.tsx):**
    Mark "use client". Build the full results UI per the prototype (lines 979-1102 of codeaudit-ai.jsx):

    1. **Header section** (per RSLT-01):
       - Folder name in mono font (`font-mono text-sm text-text-muted`)
       - Badges for auditType and depth using `<Badge>` component (depth badge gets `color="var(--accent)"`)
       - Completion stats: "Completed {date} . {duration} . ${cost}" in text-[12px] text-text-muted

    2. **Score + Severity grid** (per RSLT-02, RSLT-03):
       - Two-column grid (`grid grid-cols-2 gap-4 mb-7`)
       - Left card: `<Card>` containing `<HealthScore score={findings.summary.score} size="lg" />` + "Health Score" label + "{score} / 100" in font-mono text-[26px] + grade assessment text (>70: "Needs improvement" in warning, >40: "Critical attention needed" in destructive, etc.)
       - Right card: `<Card>` containing "Severity Breakdown" label + `<SeverityBar data={findings.summary.findings_count} />`

    3. **Cost summary banner** (per RSLT-04):
       - Surface bg card with border, flex row: left shows "Total cost: **$X.XX** (N tokens)", right shows action buttons
       - Cost = `(audit.actualCostMicrodollars / 1_000_000).toFixed(2)` -- note microdollars precision
       - Token count = audit.tokenCount formatted with `.toLocaleString()`
       - Expandable per-phase breakdown: clicking a toggle shows a list of phases with their individual tokensUsed and cost. Phase cost = `(phase.tokensUsed * (audit.actualCostMicrodollars / Math.max(audit.tokenCount, 1))) / 1_000_000` -- proportional cost estimation
       - Use `useState<boolean>(false)` for expansion toggle

    4. **Action buttons** (per RSLT-05) in the cost banner right side:
       - "Executive Report" -- `<a href={/audit/${audit.id}/executive} target="_blank">` wrapped in `<Button variant="outline" size="sm">`
       - "Technical Report" -- `<a href={/audit/${audit.id}/technical} target="_blank">` wrapped in `<Button variant="outline" size="sm">`
       - "Download All" -- `<a href={/api/audit/${audit.id}/download}>` wrapped in `<Button variant="outline" size="sm">`
       - Use Button's `asChild` prop with an `<a>` tag for proper link semantics

    5. **Filter bar** (per RSLT-06):
       - State: `useState<string>("all")` for active filter
       - Pills: "All", "Critical", "High", "Medium", "Low", "Info"
       - Each pill shows count: All shows total findings length, others show findings_count[severity]
       - Active pill styling: accent border/bg for "All", severity-specific color border/bg for others (use the severity CSS variables: `var(--severity-critical)`, etc.)
       - Inactive: transparent bg, border-border, text-text-muted
       - Pill element: `<button>` with `px-3 py-1 rounded-lg border text-[12px] font-medium capitalize cursor-pointer`

    6. **Findings list** (per RSLT-07):
       - Filter findings array by selected severity (or show all)
       - Each finding card: `div` with `bg-surface border border-border rounded-xl p-3.5 cursor-pointer` and a 3px left border colored by severity: `borderLeft: 3px solid var(--severity-${finding.severity})`
       - Inside each card:
         - `<Badge color={severityColor}>` with severity label (Critical, High, Medium, Low, Info)
         - Title in `text-[13px] font-semibold`
         - File path: `finding.filePaths?.[0]` in `font-mono text-[12px] text-accent` with optional `:${finding.lineNumbers?.[0]}`
         - Evidence/description: `finding.description` in `text-[12px] text-text-muted leading-relaxed`
         - Expandable remediation: `useState<number | null>(null)` tracks which finding is expanded. When expanded, show `finding.recommendation` in a div with `bg-elevated rounded-lg p-3 text-[12px] text-text-secondary border-l-2 border-accent` with "Remediation:" label in bold
       - Expand/collapse chevron icon (SVG arrow) on the right side

    Severity color mapping constant:

    - `formatCost(microdollars: number)`: return `$${(microdollars / 1_000_000).toFixed(2)}`
    - `formatDate(iso: string)`: return formatted date like "Mar 20, 2026"
