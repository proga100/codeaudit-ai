    - Accept `searchParams: Promise<{ a?: string; b?: string }>`, await it

    - Query both audits from DB: `db.select().from(audits).where(eq(audits.id, aId)).get()` for each
    - If either not found or either has no findings, show error or redirect to /history
    - Determine which is "previous" (older) and "latest" (newer) by comparing createdAt timestamps
    - Compute the findings diff using Set-based matching per the Phase 4 decision:
      - Composite key function: `(f: AuditFinding) => f.title + "|||" + (f.filePaths?.[0] ?? "")`
      - `prevKeys = new Set(prevFindings.map(compositeKey))`
      - `currKeys = new Set(currFindings.map(compositeKey))`
      - Resolved: findings in prev that are NOT in currKeys
      - New: findings in curr that are NOT in prevKeys
      - Persisted: findings in curr that ARE in prevKeys
    - Serialize for client:
      ```typescript
      type SerializedCompareAudit = {
        id: string; folderName: string; auditType: string; depth: string;
      ```
